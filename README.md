# BlockCraft 控制台沙盒游戏
BlockCraft 是一款基于 C++ 开发的 Windows 控制台沙盒游戏，聚焦轻量化的沙盒核心玩法（方块挖掘/放置、背包管理、存档），适配 Windows 控制台环境运行。

## 一、快速开始
### 1. 编译运行
- 环境要求：Windows 系统 + Visual Studio（支持 C++ 编译）
- 操作步骤：
  1. 克隆本仓库到本地；
  2. 双击 `BlockCraft.slnx` 打开 Visual Studio 解决方案；
  3. 编译并运行项目，自动启动控制台游戏界面。

### 2. 基础操作
| 按键/指令类型 | 操作方式                | 功能说明                     |
|---------------|-------------------------|------------------------------|
| 移动控制      | W/A/S/D 键              | 角色上下左右移动             |
| 方块挖掘      | i/j/k/l 键              | 挖掘不同方向/位置的方块      |
| 方块放置      | I/J/K/L 键（大写）      | 放置不同方向/位置的方块      |
| 背包开关      | e 键                    | 打开/关闭背包界面            |
| 物品丢弃（快捷）| q 键                   | 丢弃手中当前持有的方块       |

## 二、背包管理指令
游戏支持通过指令精细化管理背包物品，指令需在控制台交互界面输入执行，所有指令均基于**背包格子编号**（编号从 1 开始，对应背包第 N 格）：

| 指令格式                | 功能说明                                                                 | 示例                          |
|-------------------------|--------------------------------------------------------------------------|-------------------------------|
| `/swap <num1> <num2>`   | 交换背包中编号为 num1 和 num2 的物品（格子编号需有效且不重复）| `/swap 1 4`（交换1格和4格物品） |
| `/clear`                | 清空整个背包，所有格子恢复为无物品状态                                   | `/clear`                      |
| `/throw <slot>`         | 丢弃指定背包格子（slot）中的**1个**物品（格子编号需有效）| `/throw 3`（丢弃3格中1个物品） |
| `/throw_all <slot>`     | 丢弃指定背包格子（slot）中的**全部**物品（格子编号需有效）| `/throw_all 5`（清空5格所有物品） |

### 指令注意事项
1. 背包格子编号范围为 1~INVSize（INVSize 为背包总格子数，超出范围指令无效）；
2. `/swap` 指令中 num1 和 num2 不能为同一编号，否则指令执行失败；
3. 所有指令执行失败时，控制台会反馈错误提示，指令执行成功则直接生效。

## 三、存档说明
游戏存档数据存储在 `save.txt` 文件中，退出游戏时自动保存当前地图、玩家位置、背包物品等信息，重新启动游戏时自动加载存档。

## 四、技术说明
- 开发语言：C++
- 运行环境：Windows 控制台（无需图形界面库）
- 核心模块：
  - `Player.h`：玩家角色、背包指令、物品交互核心逻辑；
  - `Game.h`：游戏主循环、场景交互逻辑；
  - `VecMap.h/saveMap.h`：地图数据管理、存档读写；
  - `UI.h`：控制台界面渲染与交互。
